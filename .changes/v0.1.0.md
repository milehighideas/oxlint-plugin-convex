## v0.1.0 - 2026-06-29
### Added
* Initial release of the oxlint Convex plugin with 9 syntactic rules.
* `file-size` — flags files over `maxLines`.
* `max-functions` — flags more than `maxFunctions` registered Convex functions per file.
* `require-returns` — flags public `query`/`mutation`/`action` missing a `returns:` validator.
* `no-any-returns` — flags `returns:` containing `v.any()` (including nested in `v.union`).
* `no-api-imports` — flags importing `api` from `_generated/api` inside `convex/` (use `internal.*`).
* `type-exports-location` — flags `type`/`interface` exports outside a `types/` folder or `*.types.ts`.
* `no-filter-in-query` — flags `.filter(q => … q.field(…))` on a db query (use `.withIndex()`).
* `no-collect-in-query` — flags `.collect()` on a db query (prefer `.take()`/`.paginate()`).
* `crud-structure` — opt-in CRUD layout enforcement via `crudDomains`.
* Shared threshold resolution from the nearest `.pre-commit.json` `convexCheckConfig`.
